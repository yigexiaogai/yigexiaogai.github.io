# 团队协作和结合IDEA
上一节我们接触了 **版本穿梭、分支和远端推送** ，这说明我们已经能够在本地和远端管理项目了，接下来我们就需要在团队中合作了。

## 团队协作
团队协作其实是比较简单的一块内容，其实就是在github上邀请团队中的成员，当成员同意之后，就可以修改代码，并且推送到远端，覆盖原有项目了。
